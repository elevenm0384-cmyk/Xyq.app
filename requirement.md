# 🏗️ 增强版架构分层设计 (我要干的活)
为了体现架构设计的专业性，我们将基础后端分为以下几个模块，请在后续听到指令后逐一生成：

## 模块一：通用组件层 (common.cj)
- `ErrorCode` 枚举：统一 HTTP 与业务状态码。
- `ApiResponse<T>` 泛型类：统一 JSON 响应格式（包含 code, msg, data）。
- `PageResult<T>` 泛型类：统一分页数据结构（包含总条数、当前页、列表数据）。

## 模块二：领域模型与数据载体 (models.cj)
- **Entity（数据库实体）**：`User` (用户), `BaseItem` (统一物品), `Category` (分类)。包含 ID、创建时间等底层字段。
- **DTO（数据传输对象）**：例如 `PublishItemReq`（前端发来的发布请求体），过滤无效字段。
- **枚举**：`ItemType` (二手/失物), `Role` (学生/管理员)。

## 模块三：仓储数据层 (repository.cj)
- 无真实 DB，采用内存集合构建。
- `SessionManager`：模拟 Token 生成与校验，管理用户登录态。
- `UserRepository` & `ItemRepository`：封装基础的 `save()`, `findById()`, `findAll()` 方法，对上层屏蔽 HashMap 的底层操作。
- `MockDataInit`：提供初始化假数据的脚本函数，保证前端一运行就有满屏数据。

## 模块四：基础业务服务层 (base_service.cj)
- `AuthService`：处理注册、登录（发放 Token）、登出。
- `CategoryService`：获取分类树/列表。
- `ItemBaseService`：处理基础的发布物品、根据 ID 查详情、基础分页列表（供首页展示，复杂搜索过滤交由4号处理）。
